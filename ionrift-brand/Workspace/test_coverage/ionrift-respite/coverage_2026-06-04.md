## Why These Tests Matter

This batch targets five new service modules introduced in v3.0.0–v3.0.2 that had zero test coverage:

1. **TrainingSettings** is a brand-new subsystem (v3.0.1) that computes diminishing-returns XP using a triangular formula. A wrong coefficient or clamping bug would silently over- or under-reward players across every training rest. The tests lock the documented 15/45 penalty curve.

2. **CampGearScanner utilities** were extracted from inline checks into exported functions during the campfire ceremony refactor. They gate whether fire can be lit, how much firewood is spent, and whether a tinderbox is available. A naming regression in `isActorFirewoodItem` would break the entire Make Camp flow.

3. **RestProfileSettings** controls which UI path renders (simple vs. full, stations vs. panel, campfire minigame). Each gate is a single-setting check but the combination determines the user's entire experience. Without tests, any settings key rename silently breaks routing.

4. **ScoutingSettings** is a single-function module but it's the sole gate for the travel scouting feature. The null/undefined coercion edge cases ensure the setting is robust across fresh worlds and migrations.

5. **Double-consumption regression** re-establishes the critical regression test that was moved to the private testharness in 4cfe828. The `itemsConsumed` flag prevents inventory data loss in multi-day meal flows — the highest-impact user-facing bug fixed in v3.0.0.
