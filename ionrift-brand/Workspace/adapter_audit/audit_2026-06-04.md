## Recommended Actions

### Critical (Fix Immediately)

1. **[B3/B4] Route scouting, foraging, and hunting skill rolls through the adapter in `RestSetupApp`.**
   - Lines 3884–3885 and 12986–12993 access `actor.system?.skills?.sur`, `nat`, `prc` with DnD5e 3-letter abbreviations.
   - PF2e uses full skill names (`survival`, `nature`, `perception`). These return 0, making all outdoor activity rolls have +0 modifier on PF2e.
   - **Fix:** Replace with `adapter.getSkillTotal(actor, adapter.normalizeSkillKey("sur"))` pattern (already used in `TravelResolver._getSurvivalMod()`).

2. **[B11] Route dehydration CON save computation through adapter in `RestSetupApp`.**
   - Line 7769–7771 reads `actor.system?.abilities?.con?.mod` (works) + `actor.system?.attributes?.prof` (undefined on PF2e).
   - Dehydration saves are computed as `conMod + 0` instead of the full fortitude save bonus.
   - **Fix:** Replace with `adapter.getSaveBonus(actor, "con")` (already done in `MealDelegate`).

3. **[B1/B2] Route Arcana check through adapter in `CopySpellHandler`.**
   - Lines 162 and 359 access `actor.system?.skills?.arc` which is undefined on PF2e (correct key: `arcana`).
   - Spell copy Arcana rolls always get +0 modifier on PF2e.
   - **Fix:** Replace with `adapter.getSkillTotal(actor, adapter.normalizeSkillKey("arc"))`.
