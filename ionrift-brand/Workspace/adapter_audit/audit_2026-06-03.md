## Recommended Actions

### Critical (Fix Immediately)

1. **[R8/R9] Fix PF2e rest hooks.** The hook names `pf2e.preShortRest`, `pf2e.preLongRest`, `pf2e.preRestCompleted` do not exist in the PF2e system. Respite cannot intercept or suppress native PF2e rest recovery. **Action:** Replace with `game.pf2e.actions.restForTheNight()` call at rest completion (see C6). Since PF2e has no hookable rest flow, the adapter should signal to the rest engine that suppression is not available, and Respite should call `restForTheNight()` directly instead of relying on hook interception.

2. **[B31] Route meal buff application through the adapter.** `MealPhaseHandler._resolveBuff()` gates all buff application behind `game.system.id === "dnd5e"`. On PF2e, meals compute buffs but never apply them. Add adapter methods: `applyTempHP(actor, amount)`, `applyHealing(actor, amount)`, and route the exhaustion save through the existing `applyExhaustionDelta()`.

3. **[B8] Route `RestFlowEngine._calculateRecovery()` through the adapter.** This method reads `actor.system.attributes.hp`, `system.attributes.hd`, and `system.details.level` directly. On PF2e, `system.attributes.hd` does not exist (returns undefined), causing recovery math to produce 0 HD recovery and potentially NaN values.

4. **[B18] Fix skill proficiency check in `ActivityResolver._meetsPrerequisites()`.** Uses `actor.system.skills[s].proficient > 0` which is a DnD5e-specific field. PF2e skills use `.rank` not `.proficient`. Replace with `adapter.isSkillProficient()`.

5. **[B43/B44] Fix ActiveEffect key paths for meal buffs.** The AE keys `system.abilities.${ab}.save.roll.mode` and `system.traits.dr.value` are DnD5e-specific. On PF2e, these paths do not exist. Add an adapter method `getActiveEffectKeys(buffType)` that returns system-appropriate AE change entries. PF2e uses Rule Elements rather than ActiveEffects for most mechanical bonuses; consider whether AE-based buffs work at all on PF2e.

6. **[B28] Fix CON save computation in `MealDelegate`.** Lines 579-581 and 884-886 directly access `actor.system.abilities.con.mod`, `actor.system.abilities.con.save`, and `actor.system.attributes.prof`. On PF2e, `con.save` does not exist at that path (PF2e saves are Statistic objects at `actor.saves.fortitude`). The computed save bonus will be wrong.

### High Priority (Fix Before Next Release)

7. **[B1-B4, B9-B10, B13] Route all HP writes through the adapter.** 10+ locations write directly to `system.attributes.hp.value`. While PF2e happens to share this path, routing through `adapter.applyHPRestore()` ensures future compatibility and centralizes HP change logging.

8. **[B22-B27] Route skill/ability reads through the adapter.** `RestConstants`, `RestSetupApp`, `RollRequestManager`, `TravelResolver`, and `TravelResolutionDelegate` all read `actor.system.skills` and `actor.system.abilities` directly. On PF2e, the skill keys differ (e.g., PF2e uses `survival` not `sur`, `nature` not `nat`, `perception` not `prc`). These will silently return 0 on PF2e.

9. **[B33-B34] Add currency abstraction to the adapter.** `CopySpellHandler` and `ResourceSink` access `actor.system.currency.gp` directly. PF2e uses the same path but with different denominations and coin economy. Add `getCurrency()` and `deductCurrency()` adapter methods.

10. **[B35-B36] Route tool proficiency checks through the adapter.** `CraftingEngine` and `ActivityResolver` access `actor.system.tools` which does not exist in PF2e. Replace with `adapter.isToolProficient()`.

11. **[R4] Address `getProficiencyBonus()` returning 0 for PF2e.** Either: (a) return level-based proficiency (`level + rank`), (b) document that callers must not use this for DC scaling on PF2e, or (c) add a system-aware DC scaling method to the adapter.
